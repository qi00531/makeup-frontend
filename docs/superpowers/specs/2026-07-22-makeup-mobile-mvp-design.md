# 美妆教程跟练移动端 MVP 前端设计

## 1. 范围与优先级

本期仅建设 React + TypeScript + Vite 前端，覆盖以下四个手机端页面：

1. 视频上传页
2. 照片上传确认页
3. 视频解析页
4. 妆容适配预览页

页面结构以用户提供的四屏参考图为最高优先级。产品文档只补充参考图未表达的交互、文案、异常状态和接口边界；两者冲突时按参考图实现。本期不实现图示教程、眼部精讲、知识库、个人中心或真实后端。

## 2. 技术架构

- React + TypeScript + Vite。
- React Router 管理独立页面和浏览器返回行为。
- 页面组件只处理展示与用户操作。
- `services` 层定义上传、解析和适配结果的数据接口，并提供前端模拟实现。
- 后续联调只替换服务实现和接口地址，不修改页面组件结构。
- 共享类型集中定义，避免页面自行拼装后端数据。

路由：

- `/`：视频上传页。
- `/photo`：照片上传确认页。
- `/parsing`：视频解析页。
- `/preview`：妆容适配预览页。
- `/practice`、`/library`、`/profile`：底部导航的轻量占位页，不扩展业务功能。

## 3. 页面结构

### 3.1 视频上传页

结构顺序严格遵循参考图：

1. 页面标题。
2. 主视频上传卡片。
3. 视频要求说明。
4. 最近任务列表。
5. 固定底部导航。

选择视频后显示文件名、大小和视频类型，并允许重新选择。文件校验失败时在上传卡片附近显示明确原因。主操作进入照片确认页。

### 3.2 照片上传确认页

结构顺序严格遵循参考图：

1. 顶部标题“确认照片”。
2. 居中的照片预览区域。
3. 上传照片的价值与拍摄要求说明卡片。
4. “重新上传”和“确认上传”两个底部操作。
5. “暂时跳过”作为明确但弱化的操作。
6. 隐私用途说明。

照片要求使用中性、友好表达：正面清晰、自然光线、无遮挡。跳过后仍可继续解析，并使用默认示意脸生成适配内容。

### 3.3 视频解析页

结构顺序严格遵循参考图：

1. 顶部状态标题。
2. 居中圆形进度及百分比。
3. 纵向解析节点列表。
4. 当前阶段说明。
5. 预计剩余时间。

节点精简为参考图能够承载的四项：检查视频质量、识别妆容步骤、生成适配预览、整理关键建议。模拟解析会自动推进并进入适配预览。失败状态保留具体原因与重新解析操作。

### 3.4 妆容适配预览页

结构顺序严格遵循参考图：

1. 顶部返回按钮和标题“适配预览”。
2. 居中的妆前 / 妆后左右对比预览。
3. 妆容色彩摘要。
4. 关键适配提示列表。
5. “适合我 / 需要微调 / 不适合我”三个判断按钮。

对比预览使用同一人物、同一角度和同一尺寸的两张图叠放。左侧为“原始状态”，右侧为“化妆后”；用户横向拖动中间分隔线，可连续查看任意比例，并能将分隔线拖到两端查看完整原图或完整妆效。分隔线必须有清楚的拖动手柄和两侧文字标签。组件同时提供“看原图”和“看妆后”两个辅助操作，供键盘、读屏或不便拖动的用户完整切换。

本期按钮提供可见反馈；“需要微调”和“不适合我”不扩展新的业务页面，用提示说明该能力将在后续流程接入。

## 4. 底部导航

底部导航按参考图采用四项：

1. 首页
2. 跟练
3. 知识库
4. 我的

上传页展示导航栏并高亮首页。其余三项可点击并进入轻量占位页。照片确认、解析和适配预览属于上传任务的连续子流程，不重复展示底部导航，避免与参考图冲突。

## 5. 视觉规范

- 手机端优先，内容宽度适配常见 320px 至 430px 视口。
- 桌面端使用柔和背景承托居中的手机内容容器，不拉伸移动端结构。
- 暖白背景、深灰文字、低饱和暖红主操作色。
- 卡片使用轻边框、小阴影和克制圆角。
- 不增加营销横幅、复杂插画、渐变大背景或参考图之外的首页模块。
- 正文保证移动端可读性，交互热区不小于 44px。
- 支持安全区域和减少动态效果设置。

## 6. 前端数据与后端衔接

核心接口类型：

- `UploadVideoResult`：任务 ID、文件信息、上传状态。
- `UploadPhotoResult`：照片 ID、预览地址；跳过时照片 ID 为空。
- `AnalysisProgress`：总体进度、当前节点、节点状态、剩余秒数、失败原因。
- `MakeupPreview`：预览图、色彩摘要、妆容摘要、适配提示。

模拟服务与未来真实服务遵循相同方法签名。页面不直接依赖模拟数据常量，也不在组件内拼接未来接口 URL。

## 7. 状态与异常处理

- 未选择视频时禁止进入下一步。
- 不支持的视频格式或超限文件显示就地错误。
- 照片选择失败后允许重新上传或跳过。
- 解析失败显示原因和重试入口。
- 刷新解析页时可恢复一份前端任务状态，避免直接空白。
- 图片资源加载失败时使用不带外貌评判的中性示意占位图。

## 8. 验收标准

- 四个核心页面的模块顺序与参考图一致。
- 上传视频、选择或跳过照片、模拟解析、进入预览可完整走通。
- 上传页具有四项固定底部导航，标签与图标清楚、当前项可识别。
- 320px 至 430px 宽度无横向滚动、遮挡或底部操作被安全区域覆盖。
- 上传、解析和错误状态有明确反馈。
- 页面与服务层解耦，后续能在不改页面结构的情况下接入后端。
- 适配页对比组件可通过拖动查看任意妆前 / 妆后比例，并可完整查看两侧图片。
- 文档与参考图冲突处已全部按参考图收敛，没有加入本期范围外页面。

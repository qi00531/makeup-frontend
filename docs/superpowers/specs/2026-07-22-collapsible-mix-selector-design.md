# 混搭折叠选择器设计

## 目标

将知识库“混搭”页的五个部位选择区改为独立卡片式折叠列表，在不改变后续预制加载、妆效预览、适配判断和微调流程的前提下，减少页面初始高度并简化跳过操作。

## 页面结构

- 底妆、眼妆、腮红、修容、唇妆仍按当前顺序纵向排列。
- 每个部位使用一张独立卡片，默认全部折叠。
- 点击卡片标题区域切换该部位的展开与收起状态。
- 允许同时展开多个部位，避免用户切换卡片时丢失上下文。
- 展开后显示该部位的预制素材选项，不显示单独的“跳过此部位”选项。

## 状态展示

- 未选择时，收起卡片只显示部位名称和展开箭头，右侧不显示“默认跳过”“待选择”等文字。
- 展开状态只通过箭头方向、边框色和内容区域表达，不显示“已展开”文字。
- 已选择时，收起或展开卡片的右侧显示已选素材名称。
- 选项本身沿用当前色块、标题与来源信息；选中项使用强调边框、浅色背景和勾选图标。

## 数据与交互规则

- 五个部位的初始值均为 `null`，表示本次混搭跳过该部位。
- 用户未点击某个部位，或展开后没有选择素材，生成时都提交 `null`。
- 点击素材后，该部位保存素材 ID；点击同部位的其他素材会替换当前选择。
- 因为所有部位从初始状态起就已有明确值，进入页面后“生成效果”按钮直接可用。
- 提交的数据结构仍为完整 `MixDecision`，包含 `base`、`eyes`、`blush`、`contour`、`lips` 五个键。

## 保持不变的范围

- 不改变知识库标签栏、底部导航和页面背景。
- 不改变 `/mix/generating` 的加载可视化。
- 不改变 `/mix/preview` 的左右妆效对比。
- 不改变“适合我 → 图示教程”和“需要微调 → 微调问卷 → 图示教程”的现有逻辑。
- 不增加用户上传、素材播放或实时生成能力；所有内容仍为预制。

## 验收与测试

- 初次渲染时五张部位卡片均为折叠状态，页面中看不到素材选项。
- 点击部位标题后只展开对应选项，再次点击可收起。
- 未选择任何素材也可以生成，保存的五个部位值均为 `null`。
- 选择一个素材后，卡片显示素材名称，生成数据仅该部位为素材 ID。
- 页面不出现“默认跳过”“已展开”“待选择”或“跳过此部位”等状态文案。
- 原有加载、预览和教程流程测试继续通过。

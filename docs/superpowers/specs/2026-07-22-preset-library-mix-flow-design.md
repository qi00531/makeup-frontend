# 预制知识库与混搭生成流程设计

## 目标

知识库中的教程和部位资产均由产品方预先制作并由后端提供。用户只能浏览视频封面并进入对应的预制图示教程，不提供视频播放或用户上传能力。

混搭流程改为五个部位逐项决策。选择阶段不展示妆容效果图；用户完成全部决策后进入可视化加载页，随后查看预制效果，选择“适合我”或“需要微调”，继续沿用现有图示教程流程。

## 知识库资产

`LibraryAsset` 增加以下字段：

- `coverImage`：预制视频封面 URL。
- `tutorialId`：点击资产后打开的预制图示教程 ID。

教程和部位卡片统一使用封面区域。当前本地服务提供可替换的静态封面，后端接入后直接返回真实 CDN URL。点击任意教程或部位资产，都导航到 `/tutorial` 并携带该资产的 `tutorialId`。页面不出现上传入口、文件选择器或播放器。

## 混搭选择

混搭部位固定为：底妆、眼妆、腮红、修容、唇妆。

每个部位有三种状态：

- `pending`：尚未决定。
- `selected`：选择了一个预制资产。
- `skipped`：明确选择“跳过此部位”。

用户必须让五个部位全部离开 `pending` 状态，才能点击“生成效果”。跳过是有效决定，不等于未选择。选择页只显示部位、预制资产选项、跳过选项、完成进度和生成按钮，不显示人脸或最终妆效图片。

## 加载与结果

路由流程：

```text
/library?tab=mix
  → /mix/generating
  → /mix/preview
      ├─ 适合我 → /tutorial
      └─ 需要微调 → /adjust → /tutorial
```

`/mix/generating` 展示环形进度、阶段节点和预计剩余时间。前端本地服务使用短时确定性进度模拟；后端接入后由任务状态接口驱动。

`/mix/preview` 使用现有妆前/妆后滑动对比组件和现有两个决策按钮。页面展示由预制组合结果返回的效果图、摘要和教程 ID。“适合我”打开该结果的预制图示教程；“需要微调”进入现有问卷，并在生成教程时保留混搭结果的教程上下文。

## 数据契约

混搭选择使用五个固定键，值为资产 ID 或 `null`（跳过）：

```ts
type MixDecision = Record<'base' | 'eyes' | 'blush' | 'contour' | 'lips', string | null>;
```

生成结果包含：

```ts
interface MixResult {
  id: string;
  beforeImage: string;
  afterImage: string;
  title: string;
  summary: string;
  tutorialId: string;
}
```

本地服务按照选择内容返回确定性的预制结果。HTTP 服务接入后把选择提交给后端，由后端将组合映射到预制结果，不在浏览器或后端实时生成图片。

## 状态与异常

- 选择决定保存在路由 state 和 `sessionStorage`，刷新加载页或预览页后可恢复。
- 缺少完整选择时，加载页返回混搭选择页。
- 预制结果加载失败时显示失败原因和“重新生成 / 返回修改”操作。
- 直接访问无结果的预览路由时返回混搭选择页。

## 测试与验收

1. 知识库没有用户上传或播放器，资产展示封面并进入对应预制教程。
2. 混搭选择阶段不存在人脸效果图。
3. 五个部位必须全部选择或跳过，生成按钮才可用。
4. 生成后依次经过加载页和预制效果预览。
5. 预览的“适合我”进入对应教程，“需要微调”进入问卷并最终进入教程。
6. 手机端内部滚动、固定底部导航和现有视觉风格保持不变。
